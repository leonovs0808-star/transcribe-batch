# Инструкция для ИИ-агента: установить transcribe-batch

Ты помогаешь пользователю поставить и настроить **transcribe-batch** — скрипт,
который расшифровывает аудио/видео в текст через Groq Whisper. Пользователь,
скорее всего, не программист — веди его пошагово, без жаргона, объясняй что
и куда нажать/вставить.

## Твоя роль

Провести пользователя через 5 шагов:
1. Проверить/поставить зависимости (Python 3, ffmpeg, git).
2. Скачать проект.
3. Помочь получить бесплатный Groq API-ключ и вписать его в `.env`.
4. Сделать тестовый прогон на одном файле.
5. Запустить массовую расшифровку папки.

Работай в терминале пользователя, если у тебя есть доступ к его компьютеру
(Claude Code, аналоги). Если доступа к терминалу нет (обычный чат ChatGPT/Claude
в браузере, без выполнения команд на компьютере пользователя) — диктуй
пользователю команды по одной, жди что он скопирует и пришлёт результат.
Ни ChatGPT, ни Claude в браузере физически не видят файлы на компьютере
пользователя и не могут ничего там открыть или изменить сами — весь монтаж
идёт через то, что пользователь копирует твои команды в свой Терминал
(Mac/Linux) или PowerShell/Командную строку (Windows) и присылает вывод обратно.

**Как объяснить пользователю, что такое терминал, если он не знает:**
Mac — приложение "Терминал" (Programs → Utilities, или Cmd+Space → "Терминал").
Windows — "PowerShell" (Пуск → набрать "PowerShell").

## Шаг 1. Проверь зависимости

```bash
python3 --version
ffmpeg -version
git --version
```

**Windows:** если `python3` не найден, но `python --version` работает —
дальше по всей инструкции используй `python` вместо `python3` (замени во
всех командах ниже). Если и `python` не находится — переустановка с
python.org с галочкой "Add python.exe to PATH" при установке чаще всего чинит это.

- **Python 3 отсутствует** — направь на https://www.python.org/downloads/ (Mac/Windows)
  или `sudo apt install python3` (Linux).
- **ffmpeg отсутствует**:
  - Mac: `brew install ffmpeg` (если нет Homebrew — сначала `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`)
  - Linux: `sudo apt install ffmpeg`
  - Windows: скачать с https://www.gyan.dev/ffmpeg/builds/ (сборка `essentials`), распаковать,
    добавить папку `bin` в PATH. Либо, если есть `choco`: `choco install ffmpeg`.
- **git отсутствует** — на Mac обычно ставится вместе с Xcode Command Line Tools
  (`xcode-select --install`), на Windows — https://git-scm.com/download/win

Не продолжай следующий шаг, пока `python3 --version` и `ffmpeg -version` не
отвечают версией без ошибок.

## Шаг 2. Скачай проект

```bash
git clone https://github.com/leonovs0808-star/transcribe-batch.git
cd transcribe-batch
cp .env.example .env
```

Если `git` не хочет ставиться или пользователь не готов с ним разбираться —
альтернатива: скачать ZIP со страницы https://github.com/leonovs0808-star/transcribe-batch
(зелёная кнопка **Code → Download ZIP**), распаковать. Папка после распаковки
обычно называется `transcribe-batch-main` (не `transcribe-batch`) — уточни
точное имя у пользователя или попроси прислать `ls` (Mac) / `dir` (Windows)
из папки Downloads. Дальше зайди в неё (`cd путь/к/папке`) и обязательно
выполни ту же команду создания `.env`, что и в git-варианте:

```bash
cp .env.example .env
```

(на Windows в PowerShell — `copy .env.example .env`)

Без этого шага файла `.env` не будет и следующий шаг сломается.

## Шаг 3. Получи Groq API-ключ

1. Открой https://console.groq.com в браузере.
2. **Sign In** → войти через Google или Email.
3. Слева выбери **API Keys**.
4. **Create API Key** → любое название → **Submit**.
5. Скопируй строку `gsk_...` — показывается один раз, потом её не восстановить,
   если потеряется — просто создать новую тем же способом.

**Что дальше с этим токеном.** Он должен попасть в файл `.env` в папке проекта
(тот же, что создался командой `cp .env.example .env` на шаге 2). Это ЕДИНСТВЕННОЕ
место, куда он нужен — больше никуда вводить его не надо.

**Основной способ — одной командой в терминале** (надёжнее ручного редактирования,
`.env` — файл с точкой в начале имени, в Finder/Проводнике он скрыт по умолчанию
и новичок его не найдёт двойным кликом). Попроси пользователя вставить свой ключ
в чат, подставь его вместо `ВСТАВЬ_КЛЮЧ_СЮДА` и выполни (или продиктуй) команду
из папки проекта:

```bash
python3 -c "
import re
p = '.env'
key = 'ВСТАВЬ_КЛЮЧ_СЮДА'
s = open(p, encoding='utf-8').read()
s = re.sub(r'GROQ_API_KEY=.*', f'GROQ_API_KEY={key}', s)
open(p, 'w', encoding='utf-8').write(s)
print('OK, ключ записан в .env')
"
```

Если команда напечатала `OK, ключ записан в .env` — всё готово, дальше к шагу 4.

**Запасной способ — вручную через текстовый редактор** (если пользователь
принципиально не хочет больше трогать терминал): предупреди, что `.env` —
скрытый файл, и объясни как его показать:
- Mac Finder: находясь в папке проекта, нажать **Cmd+Shift+.** (точка) — скрытые файлы появятся.
- Windows Проводник: вкладка **Вид** → включить **Скрытые элементы**.

Дальше — открыть `.env` в TextEdit/Блокноте, заменить плейсхолдер на реальный ключ:
```
GROQ_API_KEY=gsk_реальный_ключ_пользователя
```
и сохранить файл **как обычный текст**, не меняя расширение (TextEdit может
предложить сохранить как `.env.txt` — проследи чтобы название файла осталось
именно `.env`, без `.txt` на конце).

## Шаг 4. Тестовый прогон

Попроси пользователя назвать один короткий файл (аудио или видео, лучше до
5 минут) для проверки. Запусти:

```bash
python3 transcribe.py "/путь/к/тестовому/файлу"
```

Проверь результат: рядом с файлом должен появиться `<имя>.md` с текстом внутри.
Покажи пользователю содержимое, спроси всё ли устраивает (язык, качество).

Если упало с ошибкой — смотри раздел "Если что-то не работает" ниже.

## Шаг 5. Массовая расшифровка

Спроси путь к папке с материалами. Сначала покажи план (что найдётся, без
запуска):

```bash
python3 transcribe.py "/путь/к/папке" --plan
```

Покажи список пользователю, подтверди что это то, что нужно. Затем запусти
реальную расшифровку:

```bash
python3 transcribe.py "/путь/к/папке"
```

Для больших папок (много файлов/долгих видео) это может занять часы — предупреди
пользователя и, если у тебя есть доступ к терминалу, запусти в фоне:

```bash
nohup python3 transcribe.py "/путь/к/папке" > /tmp/transcribe.log 2>&1 &
```

и периодически смотри `tail -20 /tmp/transcribe.log`.

## Разделение по спикерам (опционально)

Если в материалах диалоги/интервью, где важно кто что сказал — предложи
дополнительно завести ключ Deepgram:

1. https://console.deepgram.com → регистрация → **API Keys** → создать ключ.
2. Вписать в `.env`: `DEEPGRAM_API_KEY=твой_ключ`.
3. Запускать с флагом `--speakers`:
   ```bash
   python3 transcribe.py "/путь/к/папке" --speakers
   ```

Это отдельный платный/бесплатный лимит от Deepgram, не путать с Groq.

## Если что-то не работает

- **"ffmpeg извлечение аудио упало"** — проверь `ffmpeg -version`, переустанови если нужно.
- **"GROQ_API_KEY не найден"** — ключ не вписан в `.env` или файл `.env` не рядом со скриптом.
- **"Groq не ответил"** — проверь интернет; если пользователь в стране, где Groq
  заблокирован (например РФ) — понадобится прокси:
  ```bash
  HTTPS_PROXY=http://127.0.0.1:порт python3 transcribe.py "/путь"
  ```
  порт зависит от того, каким VPN/прокси пользуется пользователь — уточни у него.
- **Текст на английском вместо русского** — Groq сам определяет язык по содержимому,
  обычно не требует настройки; если стабильно путает — сообщи автору (см. README).
