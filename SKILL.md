# transcribe-batch — расшифровка аудио/видео

## Где лежит скрипт

`transcribe.py` — рядом с этим файлом SKILL.md, в этой же папке скила.

## Что делает

- Один файл или папка (рекурсивно): `.mp4 .mov .mkv .webm .avi .m4v .mp3 .m4a .aac .wav .ogg .opus`
- Кладёт результат в `<имя файла>.md` рядом с исходником
- Уже расшифрованные файлы пропускает — safe перезапускать
- `--speakers` — разделение по спикерам через Deepgram (для диалогов/интервью)

## Перед первым запуском

Проверь, что настроено:
1. `ffmpeg -version` — если не найден, попроси установить (`brew install ffmpeg` / `apt install ffmpeg`)
2. В папке скила есть `.env` с `GROQ_API_KEY=...` — если нет, скопируй `.env.example` в `.env`
   и попроси пользователя вставить ключ с [console.groq.com/keys](https://console.groq.com/keys)
3. Для `--speakers` нужен ещё `DEEPGRAM_API_KEY` в том же `.env`
   ([console.deepgram.com](https://console.deepgram.com/))

## Как вызывать

Один файл:

```bash
python3 "<путь_к_папке_скила>/transcribe.py" "/абсолютный/путь/к/файлу.mp4"
```

Вся папка:

```bash
python3 "<путь_к_папке_скила>/transcribe.py" "/абсолютный/путь/к/папке"
```

Только план (что будет обработано, без запуска):

```bash
python3 "<путь_к_папке_скила>/transcribe.py" "/путь" --plan
```

С разделением по спикерам:

```bash
python3 "<путь_к_папке_скила>/transcribe.py" "/путь" --speakers
```

Если у пользователя Groq заблокирован в сети — добавь прокси перед командой:
`HTTPS_PROXY=http://127.0.0.1:7897 ALL_PROXY=http://127.0.0.1:7897 python3 ...`
(порт уточни у пользователя, это не универсальное значение).

## Прогресс для длинных прогонов

Запускай в фоне через `nohup ... > /tmp/transcribe_XXX.log 2>&1 &`, дальше следи
через `tail -f` / `grep -E "готово за|ОШИБКА|Итог"`.

## Отчёт

После запуска сообщи: сколько файлов найдено, какой движок (Groq/Deepgram).
После завершения: сколько готово, пропущено, ошибок, куда легли `.md`.
